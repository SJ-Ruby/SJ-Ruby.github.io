一、Ubuntu/Linux磁盘文件

​ ls /dev/sd* 文件，此类文件是磁盘设备文件，并不能直接访问磁盘，必须要将磁盘挂载到某一个目录下才可以访问。

​ 当插入磁盘设备文件（例如：U盘），会多出两个文件，其中xxx表示U盘，xxx1表示U盘的第一个分区。

df：用于显示linux系统上的文件系统的磁盘使用情况统计。

du：显示目录或文件的大小。

​ du -h –max-depth=n n为查看几级目录

二、Ubuntu/Linux文件权限

文件权限是指不同的用户或用户组对某个文件拥有的权限，文件的权限分为三组：

r：读

w：写

x：可执行

-rw-r–r– 第一位表示文件类型，剩下的每三位表示一组权限。分别对应拥有者权限(Owner)、拥有者所在组权限(Group)、其它用户权限(Other Users)。

修改文件权限命令 chmod

权限 二进制 八进制
r 100 4
w 010 2
x 001 1

权限所对应的八进制数字是每个权限对应的位相加，比如 rwx 就是4+2+1=7。

修改文件所属用户 chown

三、Linux 链接方式

​ Linux有两种连接方式：符号链接（软链接）和硬链接；符号链接相当于Windows下的快捷方式，硬链接通过文件系统的 inode 链接来产生新文件名，而不是产生新文件。

inode：记录文件属性，一个文件一个inode。inode相当于文件 ID，查找文件的时候要先找到 inode，然后才能读出文件的内容。

ln 命令用于创建链接文件。

  • 硬链接

    硬链接是多个文件都指向同一个 inode

    1. ​ 具有相同的 inode 的多个文件互为硬链接文件，创建硬链接相当于文件多了入口。
    2. 对于硬链接文件，只有删除了源文件以及对应的所有硬链接文件，文件实体才会被删除。
    3. 根据硬链接文件的特点，我们可以通过给文件创建硬链接方式来防止文件误删除。
    4. 不论修改源文件还是链接文件，另一个文件的数据都会被改变。
    5. 硬链接不能跨文件系统。
    6. 硬链接不能连接到目录。

  • 符号链接(软链接)

    符号链接类似 Windows 下的快捷方式，符号链接也叫做软链接，软链接要用的多。符号链接相当于创建了一个独立的文件，这个文件会让数据读取指向它链接的哪个文件的文件名。

    1. 可以链接到目录。

    2. 可以跨文件系统。

    3. 删除源文件以后，软链接文件也就”打不开了“。

    4. 软链接文件通过 -> 来指示具体的链接方式。

    5. 要使用绝对路径，否则链接出问题

四、make 和 makefile 文件的引入

1.make 工具

利用make工具自动完成编译工作，这些工作包括：

  • 如果修改了某几个源文件，则只重新编译这几个源文件
  • 如果某个头文件被修改了，则重新编译所有包含该头文件的源文件

利用make可以大大简化开发工作，避免不必要的重新编译。make工具通过Makefile文件来完成并自动维护编译工作，Makefile文件描述了整个工程的编译、链接规则。

2.Makefile 文件

  • TARGER(目标)：目标程序产生的文件，如可执行文件和目标文件，目标也可以是要执行的动作。
  • DEPENDENCIES(依赖文件集合)：依赖是用来产生目标的输入文件列表，一个目标通常依赖多个文件。
  • COMMAND(命令)：命令是 make 执行的动作，每个命令行的起始字符必须为 TAB 字符。
  • 如果 DEPENDENCIES 中有一个或多个文件更新的话，COMMAND 就要执行，这就也是 Makefile 最核心的内容。

​ 模式规则中，至少在规则的目标定义中要包含 “%”，否则就是一般规则，目标中的 “%” 表示对文件名的匹配，“%” 表示长度任意的非空字符串，比如 “%.c” 就是所有的以 .c 结尾的文件，类似与通配符，a.%.c 就表示以 a. 开头，以 .c 结束的所有文件。

​ 模式规则中，目标和依赖都是一系列的文件，每一次对模式规则进行解析的时候都会是不同的目标和依赖文件，而命令只有一行，使用自动化变量生成一行命令来从不同的依赖文件中对应的目标。所谓自动化变量就是这种变量会把模式中定义的一系列的文件自动的挨个取出，直至所以的符合模式的文件都取完。不二传感器 编码器 光电传感器

七、Ubuntu 交叉编译器

交叉编译器：在一种计算机环境中运行的编译程序，能编译出在另外一种环境下运行的代码。

在 Linux 下进行 C 程序开发， 我们使用的 gcc 编译器进行代码的编译， 编译完成，直接在 X86 架构下的 PC 下运行的。 而 arm 架构的， 所以我们需要使用支持 arm 的编译器并且该编译器在 X86 架构下运行， 这个编译器我们通常称为交叉编译器。
