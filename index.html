初识 git

git

git 是一个免费的开源 分布式版本控制系统，旨在快速高效地处理从小型到大型项目的所有内容。

  • 克隆 Git 资源作为工作目录。

  • 在克隆的资源上添加或修改文件。

  • 如果其他人修改了，你可以更新资源。

  • 在提交前查看修改。

  • 提交修改。

  • 在修改完成后，如果发现错误，可以撤回提交并再次修改并提交。

Git 工作区、暂存区和版本库

  • 工作区：在电脑里能看到的目录。

  • 暂存区：英文叫 stage 或 index。一般存放在 .git 目录下的 index 文件（.git/index）中，所以暂存区有时也叫作索引（index）。

  • 版本库：工作区有一个隐藏目录 .git，这个不算工作区，而是 Git 的版本库。

  • 图中的 objects 标识的区域为 Git 的对象库，实际位于 “.git/objects” 目录下，里面包含了创建的各种对象及内容。

  • 当对工作区修改（或新增）的文件执行 git add 命令时，暂存区的目录树被更新，同时工作区修改（或新增）的文件内容被写入到对象库中的一个新的对象中，而该对象的ID被记录在暂存区的文件索引中。

  • 当执行提交操作git commit时，暂存区的目录树写到版本库（对象库）中，master 分支会做相应的更新。即 master 指向的目录树就是提交时暂存区的目录树。

  • 当执行 git reset HEAD 命令时，暂存区的目录树会被重写，被 master 分支指向的目录树所替换，但是工作区不受影响。

  • 当执行 git rm –cached 命令时，会直接从暂存区删除文件，工作区则不做出改变。

  • 当执行 git checkout . 或者 git checkout – 命令时，会用暂存区全部或指定的文件替换工作区的文件。这个操作很危险，会清除工作区中未添加到暂存区的改动。

  • 当执行 git checkout HEAD . 或者 git checkout HEAD 命令时，会用 HEAD 指向的 master 分支中的全部或者部分文件替换暂存区和以及工作区中的文件。这个命令也是极具危险性的，因为不但会清除工作区中未提交的改动，也会清除暂存区中未提交的改动。

How to build a hexo blog

Posted on 2020-11-30

简介

Hexo是一个功能强大的博客框架，可使用Markdown编写，Hexo将自动生成静态文件。

一、环境配置

  • Node.js

  • git

二、安装Hexo

三、建立

文件夹内容

_config.yml 网站配置文件

package.json 应用程序数据。EJS、Stylus和Markdown渲染器是默认安装的。

scaffolds 脚手架文件夹。当创建一个新的帖子时，Hexo会将新文件建立在脚手架上。

source 源文件夹。放置网站内容的地方。Hexo 忽略隐藏的文件和名称前缀为 _（下划线）的文件或文件夹 - _posts 文件夹除外。可渲染的文件（如Markdown、HTML）将被处理并放入公共文件夹，而其他文件将被简单复制。

themes 主题文件夹。Hexo通过将网站内容与主题相结合，生成一个静态网站。、

四、配置

可以在**_config.yml**中修改大部分的配置，详见https://hexo.io/zh-cn/docs/configuration.html

五、指令

init

初始化一个网站。如果没有提供文件夹，Hexo 将在当前目录下建立一个网站。

new

创建一个新的文章。如果没有提供布局layout，Hexo 将使用 _config.yml 的默认布局。如果标题title包含空格，用引号将其包围。

默认情况下，Hexo 将使用标题来定义文件的路径。对于页面，它将创建一个该名称的目录，并在其中创建一个index.md文件。使用 –path 选项可覆盖该行为并定义文件路径。

将创建source/about/me.md文件，并在前面设置标题 “关于我”。

PS:标题是强制性的。例如，这将不会导致你所期望的行为。

generate

生成静态文件。

publish

发布草案。

server

启动一个本地服务器。默认情况下，这是http://localhost:4000/。

deploy

部署网站。

render

渲染文件。

migrate

从其他博客系统迁移内容。

clean

清理缓存文件（db.json）和生成的文件（公开）。

list

列出所有路线。

safe

禁用加载插件和脚本。如果在安装新插件后遇到问题，可以试试这个。

debug

将详细的信息记录到终端和 debug.log。

draft

显示草稿的帖子（存储在source/_drafts文件夹中）。

六、写作

要创建一个新的帖子或新的页面，可以运行以下命令。

LAYOUT
Hexo 中有三种默认布局：帖子post、页面page和草稿draft。每种布局创建的文件都会保存到不同的路径。新创建的帖子保存在source/_posts文件夹中。

FILENAME

默认情况下，Hexo 使用帖子标题作为其文件名。可以编辑 _config.yml 中的 new_post_name 设置来更改默认文件名。
